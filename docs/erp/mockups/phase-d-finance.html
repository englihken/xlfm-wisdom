DESIGN DECISIONS（本稿基线）

  • 满额即止 cap 移除（Ken 2026-07-08）：不设自动上限。改为①中心级「暂停收款」手动开关（财政判断本月已足→members 自查页显示"本月已满，感恩 🙏"）＋②透明化展示（已收 vs 支出 vs 结余→盈余互助），以透明代替强制。
  • 认捐额随人（Melaka 台账实证：RM50/月、RM200/月、RM600/年并存）：每位会员存 pledge（金额+周期），默认 RM50/月。缴费记录存明确覆盖月份区间（from–to），绝不从金额÷50推导。
  • 收据号 = 中心顺序号簿（0000963→0001028 式）：系统按中心自增预填，可手动改（衔接旧收据簿）。
  • 渠道：现金 / 银行转账 / 汇至总会 — 全部手动录入。线上支付（FPX/TNG）留 LATER。
  • 伦理红线不变：缴费自愿、永不 gate 会籍；无"逾期/欠款"概念（只有"付至 YYYY-MM"事实）；已豁免一等公民。
  • 权限：财务总监 all_centers · 中心财政 own_center（复用 centre_scope_allows）；理事会只见聚合视图（SECURITY DEFINER），永不见个人行级数据。会员自查 = 匿名+手机号验证，仅见自己（复用 C1 masked-identify 模式）。
  • 历史台账导入（Melaka xlsx 等）：走 A5 staging pipeline，新 batch type；非会员赞助者建为 member_type='member'。
  • 无 DELETE：错录收款 = 作废（voided，留审计）；writeAudit 每笔 mutation。
